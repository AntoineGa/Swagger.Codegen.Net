How to use

Commandline tool

The commandline tool generatess C# client code for a Swagger enabled API, similar like what wsdl.exe does for webservices. Provides the same functionality offered by the awesome swagger-codegen but without its prerequisites 

swagger.exe --url http://petstore.swagger.wordnik.com/api/api-docs

Options

Option Description
-u or --url The url of the API which serves the resource listing
-a or --apiname The name of the API
-n or --namespace Specifies the namespace for the generated proxy or template. The default namespace is the global namespace.
-o or --out Specifies the file (or directory) in which to save the generated proxy code. You can also specify a directory in which to create this file. The tool derives the default file name from the API name.

Examples

swagger.exe --url http://petstore.swagger.wordnik.com/api/api-docs
